Node的优势

发表于 2015-12-03   |   分类于 -Node学习   |  

随着移动互联网的涌来，同时也伴随着越来越多的新技术和思潮的到来。Node就是其中之一，
其基于Google Chrome的V8引擎，作用于服务器的解决方案。

与传统的服务器静态语言Java/C#不同，node采用的是动态语言JavaScript，不用事先编译成字
节码，V8引擎让其直接解释成本地机器码执行。如果除去静态语言强类型声明等所带来的稳定性
外，但执行性能上应该要比java好。

Node采用基于异步I/O的事件轮询，这种方式有点像GUI编程（界面编程）的事件机制，只有
当你点击某个控件时，才能触发相应的事件方法。假如现在有某个控件的事件处理方法因为数
据的加载或其他原因而延迟执行了，这时你点击另外一个控件，相对应的事件方法依然会执
行，而不会等待前一个事件方法执行完再执行。

代码规范

发表于 2015-10-14   |   分类于 -code   |  

IT虽然没有建筑学，金融那样悠久的历史，但作为社会工程一个新的分支，依然需要严谨的思
维方式。这种严谨的方式最直观的就体现在代码规范上。

软件项目一般都是团队开发，如果没有一套统一的代码规范，到了后期，项目就会看上去比较
杂乱。所以比较好的公司，一进去并不是让你立刻写代码或熟悉业务流程，而是先发一份代码
规范给你。

笔者因为之前接手过别人项目，体会过由于无代码规范所带来的那种烦躁心理，所以记录下代
码编写时需要注意的规范问题。

一：代码格式

1：类格式
类里面 公有方法，私有方法，私有成员变量，公有成员变量，静态公有方法，静态私有成员，静态公有成员，枚举类型等的一些格式定义。

2：大括号最好独占一行，并且if，for，do，while等条件语句的执行体无论多少局，最好都加上{}。

多个函数之间须添加空行，getter/setter函数之间可不加，保持紧凑。

二：归一化

代码上下文一致性
尽量保持修改点上下代码命名规则、格式、逻辑上下文的一致性，要求多人维护的代码，看起来像一个人写的一样;

每个方法体最好不要超过30行。

不同作用域的命名：
1.包名：一个功能性单词，全部小写。
2.类名： 一个或者多个单词组成，每个单词第一个字母大写ClassName。
3.接口名：I+一个或者多个单词，IPanel。
4.成员变量：一个或者多个单词组成，每个单词第一个字母小写name。
5.功能性函数名：动词+名词，首字母小写，其它单词首字母大写，changeValue, updateData。
6.函数体内局部变量：int i = 0; int x = 0;
7.事件处理函数：on+名+动(时态) onValueChanged。
8.回调函数成员变量：动词+handler，actionHandler。

常用前后缀及使用场景：
is: Boolean型状态或函数 isAdded()。
has: Boolean型状态或函数，hasChanged()。
do: 间隔调用行为函数，doUpdate()。
delay: 延时调用行为函数，delayCheckData()。
handler: 回调函数变量后缀，actionHandler()。
utils: 静态工具类后缀，StringUtils。
check: 检查密集的函数前缀，checkGuideTask()。
from: 反序列化函数前缀，fromBytes(), fromJSONObject()。
to: 序列化函数前缀，toBytes(), toJSONObject(), toXML(), toString()。
ble: 可…的，如：movable()，removable()。

三：注释

一般情况，源程序的有效注释量需达到30%，但注释的文字提倡精简话。
在多人协作开发时，文件的注释特别重要。

音频，视频

发表于 2015-09-17   |   分类于 -多媒体   |  

视频编码格式：H.264、Theora（免费）、VP8（google开源免费）。
音频编码格式：ACC、MP3、Vorbis（开源免费无专利）。

HTML5支持的视频格式：
Ogg = Theora视频编码+Vorbis音频编码的Ogg文件。（支持的浏览器：F，C，O）
MEPG4（MP4） = H.264视频编码+ACC音频编码的MPEG4文件。（支持的浏览器：S，C）
WebM = VP8视频编码+Vorbis音频编码的WebM文件。（支持的浏览器：I，F，C，O）
